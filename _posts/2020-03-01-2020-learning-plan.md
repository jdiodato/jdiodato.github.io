Inspired by [@radhikamorabia](https://twitter.com/radhikamorabia)'s blog post on her [2020 Learning Plan](https://radhika.dev/planning-2020/), I thought I'd follow suit and brainstorm my own learning road map for 2020. Specifically this post will cover my learning goals for 2020 as they relate to technology, computer programming, etc. I'd rather set a few specific goals that I can reasonably achieve rather then go overboard and miss several goals or feel pressured to achieve them all. I can always add more to the below list, and I think that'll be the best approach to developing and refining my learning plan.

## Python 
[Python](https://www.python.org/) has been the bread and butter of my programming re-education, and it's likely to remain one of my favorite programming languages for a number of reasons. At the time of making this plan, I've worked through most of the programming basics (functions, logical structures, etc) using [John Zelle's Python book](https://mcsp.wartburg.edu/zelle/python/). I really enjoy the Zelle text because it's not *just* the mechanics of learning Python. It really does read more like an introductory computer science text that *happens* to use Python as the language of choice for implementation. Once I finish with the Zelle book, I'll progress to [Automate the Boring Stuff with Python](https://automatetheboringstuff.com/) and then finishing my introduction to Python with [Learn More Python 3 The Hard Way](https://www.amazon.com/Learn-More-Python-Hard-Way/dp/0134123484). I'm also taking an intro to Python class at the moment, and it's been a great experience to learn from someone with that level of expertise. One of my main goals with Python at the moment is to create a few scripts that will automate certain tasks in my professional career as a higher education administrator.

### Godot: Video Game Development

I hope to create my first video game using the [Godot game engine](https://godotengine.org/). Realistically, it'll be a pretty straightforward 2D-platformer with what I hope will turn out to be somewhat decent pixel art! Godot projects are typically created using a programming language called [GDscript](https://docs.godotengine.org/en/3.1/getting_started/scripting/gdscript/gdscript_basics.html#doc-gdscript). While this high-level programming language *looks* similar to Python syntax, I'm told that there are actually quite a few differences! Either way, it's Python-esque (at least to my current understanding), and I'll include it here as a subset of my Python goals.

## Web Development (HTML, CSS, and Javascript)
I've wanted to learn some basic web development for a while, and I think that 2020 is a good year for me to do so. I'll likely start with [The Odin Project](https://www.theodinproject.com/home) (TOP), as I've only heard amazing things about it. The platform essentially compiles the best of the freely available web development resources and delivers it to the user with a healthy dose of practice projects to hone the skills covered in each track or module. I'm all for free/open learning resources and I'm excited to see what TOP has in store! I'll be following the [full-stack Javascript curriculum](https://www.theodinproject.com/tracks/2), which includes the following courses:
- Web Development 101
- [Javascript](https://developer.mozilla.org/en-US/docs/Web/JavaScript)
- [HTML](https://en.wikipedia.org/wiki/HTML) and [CSS](https://en.wikipedia.org/wiki/Cascading_Style_Sheets)
- [Node.JS](https://nodejs.org/en/)

I'll likely go on to further refine my Javascript skills, and possibly toss in some [PHP](https://www.php.net/) as well. I'm not terribly enthused about PHP, but I do want to dabble in Wordpress development so it's really a no-brainer. I hope to cultivate a solid understanding of vanilla Javascript before branching out and exploring a few of the various JS frameworks.
## Java
[Java](https://www.oracle.com/java/) isn't a huge priority for me at the moment, but I *am* interested in tutoring introductory computer science courses when I return home to New York City this summer. I've taken a class on Java before and I'm feeling pretty good about brushing up on Java given that I have most of the foundational concepts down from my learning of Python. My current goal for Java is to feel comfortable with the material typically covered in an introductory programming class. Between Java and Python, I should be able to work with students in most introductory programming classes. Java is also a good language to ultimately prepare me to work with [C#](https://en.wikipedia.org/wiki/C_Sharp_(programming_language)).

## Discrete Math
[Discrete math](https://en.wikipedia.org/wiki/Discrete_mathematics) is an incredibly important topic for just about anyone that works with code, and I'll be using [Discrete Mathmatics: An Open Introduction](http://discrete.openmathbooks.org/dmoi3.html). Not only is the text freely available, I appreciate that it comes with what appears to be quality practice problems and solutions. Realistically, I hope to at least make it through chapter 3 this year. That would cover the following topics:
- counting
- sequences
- symbolic learning and proofs

## Closing Thoughts
With that all being said, I think I have a pretty decent array of projects and technologies that I want to dip my hands into. I'm currently in the early stages of the [#100DaysOfCode challenge](https://www.100daysofcode.com/), and I'm excited to see what that'll do for my productivity. I'm just now getting around to codifying this list, but I've already made great progress on the Python front. I'm excited to see what else will come of this learning endeavor! I'll be regularly uploading new projects on my [GitHub](https://github.com/jdiodato), and I hope that you'll consider following along.
